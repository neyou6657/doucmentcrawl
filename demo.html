🎯 元素选择器功能演示

欢迎使用新的元素选择器功能！

这个功能让你可以像使用浏览器开发者工具一样，直接用鼠标点击选择页面上的元素。 不再需要手动输入复杂的 CSS 选择器了！

🚀 使用步骤：
  1. 点击控制面板中的 🎯 选择链接区域 按钮
  2. 移动鼠标到左侧边栏（会看到橙色高亮）
  3. 点击侧边栏区域
  4. 点击 🎯 选择内容区域 按钮
  5. 移动鼠标到主内容区并点击
  6. 现在可以点击 扫描链接 开始工作了！

✨ 主要特性

  • 可视化选择：像开发者工具一样，鼠标悬停时实时高亮元素
  • 智能识别：自动生成最优的 CSS 选择器
  • 即时反馈：显示将会匹配的链接数量
  • 简单易用：无需了解 CSS 选择器语法
  • 快捷操作：按 ESC 键可随时取消选择

📝 工作原理

当你点击"选择链接区域"按钮后：

  1. 页面会出现一个半透明遮罩层
  2. 移动鼠标时，当前元素会被橙色边框高亮
  3. 屏幕上方会显示一个黑色提示框，显示元素信息和将使用的选择器
  4. 点击元素后，工具会自动生成选择器（例如：.sidebar a）
  5. 弹出提示显示找到了多少个链接
💡 提示：
  • 选择链接区域时，尽量选择包含所有导航链接的容器元素（如侧边栏、导航栏）
  • 选择内容区域时，选择主要的文章或文档内容区域
  • 如果第一次选择不满意，可以重新选择，新的选择会覆盖旧的

🎨 视觉效果

选择模式下你会看到：

  • 橙色高亮边框：当前鼠标悬停的元素
  • 黑色提示框：显示元素标签名、类名和生成的选择器
  • 屏幕中央提示：提示你当前正在选择什么（3秒后自动消失）
  • 十字光标：鼠标光标变为十字，表示处于选择模式

🔍 智能选择器生成

工具会尝试生成最简洁有效的选择器，优先级如下：

  1. ID 选择器（如果元素有唯一 ID）
  2. 类选择器（如果类名唯一）
  3. 标签+类选择器
  4. 属性选择器（role、data-* 等）
  5. 路径选择器（作为后备方案）

选择链接区域时，会自动在选择器后面加上 a， 这样就能直接选中该区域内的所有链接了！

🎓 示例场景

场景 1：文档网站

  • 链接区域：选择左侧侧边栏 → 生成 .sidebar a
  • 内容区域：选择主内容区 → 生成 .main-content

场景 2：博客网站

  • 链接区域：选择导航菜单 → 生成 nav a
  • 内容区域：选择文章内容 → 生成 article

场景 3：API 文档

  • 链接区域：选择 API 目录 → 生成 .toc a
  • 内容区域：选择 API 描述区 → 生成 .api-doc
